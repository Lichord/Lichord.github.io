    • 1.二维数组的查找

    在一个二维数组中（每个一维数组的长度相同），每一行都按照从左到右递增的顺序排序，每一列都按照从上到下递增的顺序排序。请完成一个函数，输入这样的一个二维数组和一个整数，判断数组中是否含有该整数。

    思路：从左下角或者右上角开始查询，假如从左小角开始，如果要查询的数字大于数组元素，数组向左移动，小于则向上移动，找到则返回true，直到移动到边界(行= =0,列==数组列数）。数组的行数=array.length-1;
    数组的列数是array[0].length-1;

    3.从尾到头打印列表

    输入一个链表，按链表值从尾到头的顺序返回一个ArrayList。

    思路1：将链表的数遍历全部放入栈，再将栈pop放入ArrayList中

    思路2：通过递归，本质也是通过栈.

    5.用两个栈实现队列

    用两个栈来实现一个队列，完成队列的Push和Pop操作。 队列中的元素为int类型。

    思路1：我想到的，但是开销比较大，不建议使用。思路就是栈1需要进入的时候看看栈2是否为空，空则可以进入，不空就将栈2全部压进栈1，再push新值；栈2需要pop的时候看看栈1是否为空，不为空就先压入栈2，再pop栈2.

    思路2：栈1需要push,就用来push，栈2需要pop的时候，先pop,如果栈2为空了，就将栈1全部压进栈2，再pop。同时考虑一下如果双栈都空要pop的异常情况，以及为了稳定性，在栈1元素的数量达到某个阈值时将栈1压入栈2

    6.旋转数组中的最小数字

    把一个数组最开始的若干个元素搬到数组的末尾，我们称之为数组的旋转。 输入一个非减排序的数组的一个旋转，输出旋转数组的最小元素。 例如数组{3,4,5,1,2}为{1,2,3,4,5}的一个旋转，该数组的最小值为1。 NOTE：给出的所有元素都大于0，若数组大小为0，请返回0。

    思路1：我的思路，这个题有简便的算法，只是要考虑的情况比较多。而且这个题主要是考察二分查找，这个思路算是走了个捷径。旋转之后，一般情况下，只要前者减去后者大于0，那么后者就是最小值。要考虑数组长度为0，1，2的情况。还有考虑数组中每个数字相等，以及完全倒转之后是一个递减数组。

    思路2：二分查找

    数值相关

    数值的整数次方

    给定一个double类型的浮点数base和int类型的整数exponent。求base的exponent次方。

    保证base和exponent不同时为0

    分析

    重点在于考虑齐全各种情况

    • base!=0 && exp>0 : 乘方
    • base==0 && exp>0 : 0
    • base!=0 && exp==0 : 1
    • base==0 && exp==0 : 不考虑
    • base!=0 && exp<0 : 1÷乘方
    • base==0 && exp<0 : 0不能为分母，0的负次方无意义，报错

      优化

    • 直接累乘的时间复杂度是O(N)
    • n为偶数，a^n=a^n/2a^n/2;n为奇数，a^n=（a^（n-1）/2）（a^（n-1/2））*a,每次递归可以减小一般，算法复杂度O(logN)
