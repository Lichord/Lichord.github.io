使用maven搭建分层的聚合项目

项目结构：

  1. 一路按照默认创建一个maven项目

  2. 在pom.xml中加入

  3. 在项目下创建名为common的maven module，在该模块的pom.xml中导入依赖，后面的模块都会间接的依赖于common

  4. 创建名为pojo的模块,将common模块导入进去

版本这里可能会报错，在File->Project Structure->moudles下，选择pojo模块+Module Dependency将common依赖导入就行

  1. 同理创建名为mapper的模块,将pojo模块导入进去,创建service，导入mapper，创建miniapi模块，导入service

Mybatis逆向生成

  • 项目src/main/java文件夹下新建mapper,pojo，util文件夹

  • 项目resource文件夹下新建generatorConfig.xml配置文件

  • util文件夹下新建GeneratorDisplay类

  • 在utils下新建MyMapper类

  • pom.xml中导入包

  • 运行GeneratorDisplay的main方法

  • 将生成的pojo下的文件复制到pojo模块下的bjfu.lixian.pojo文件夹

  • 将生成的mapper的文件复制到mapper模块下的bjfu.lixian.mapper文件夹

  • 在common模块下新建utils文件夹，把utils文件夹下的MyMapper类放在文件夹下,之后的工具类都会放在这个文件夹

  • 插播之前application.xml里的配置，根据实际情况调整

开发注册用户的接口

  1. common模块，bjfu.lixian.utils文件下JSONResult类，用于返参，接口调用之后把一些信息包装成json返回

  2. MD5Utils类，对密码进行md5加密

  3. miniapi模块下bjfu.lixian新建controller文件夹，新建RegistLoginController,实现简单注册功能

  4. 在controller类中注入了UserService并用到了service的两个方法，接下来在service模块中创建bjfu.lixian.service及bjfu.lixian.service.Impl包，分别创建service接口及实现类

  • 这里用到了自动生成id的技术而不是数据库自增创建id,把org.n3r.idworker这个包复制到common模块的java文件夹下
  1. 修改并运行Application类，注入的Service类已经用@Service声明了，Service中注入的mapper类会在application中扫描包@MapperScan(basePackages= {"bjfu.lixian.mapper"})
    @ComponentScan(basePackages= {"bjfu.lixian","org.n3r.idworker"})

    能够成功运行就算ok了

    上传视频

    业务流程：用户选择视频（10秒限制）->打开选择bgm页面->选择/不选bgm,输入视频描述->Controller上传视频->保存视频截图作为封面->判断用户是否选择bgm->合并视频与bgm为新视频进行保存/直接保存视频

ffmpeg 视音频处理工具

  • 主页：http://ffmpeg.org/
  • https://www.jianshu.com/p/7ed3be01228b

    应用场景

  • 播放器
  • 转码工具：格式工厂，剪辑工具
  • 直播,视频加码，滤镜，水印，特效

    命令

  • -i选取视音频
  • -t时间，单位s
  • -y覆盖
  • 转码 C:\ffmpeg\bin>ffmpeg.exe -i C:\Users\HASEE\Videos\test.mp4 winter.avi
  • 视音频合并C:\ffmpeg\bin>ffmpeg.exe -i test.mp4 -i bgm.mp3 -t 7 -y 新视频.avi
